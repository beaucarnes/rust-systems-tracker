## 💡 The Concept

Mid-roll pivot — you break from the Rust tutorial naturally with:  
> *"Before we continue, I want to show you something that's been saving me a lot of time..."*

The demo task is **Rust-developer-relevant**: analyzing GitHub issues and writing a status update. Developers trust tools that solve their actual problems — not generic demos.

## 🎬 Screen Recording Guide

Follow these steps **in order** while recording your screen. Each step maps to a moment in the script.

### Step 1 — Open MindsHub Cowork *(0:00 – 0:05)*
- Launch the MindsHub desktop app
- Show a clean workspace — no clutter, dark theme visible
- **Say:** *"Before we jump back in — quick word from today's sponsor, MindsHub AI."*

### Step 2 — Show Connected Apps Panel *(0:05 – 0:15)*
- Open the **Connected Data Sources** or integrations panel
- Make sure **GitHub** is visibly connected
- Let the viewer see it for 2–3 seconds — this is a key trust signal
- **Say:** *"MindsHub is a desktop AI workspace... You connect your tools once..."*

### Step 3 — Type the Demo Task *(0:15 – 0:25)*
- In the main chat or task input, type this prompt (or dictate):
  > *"Look at the open issues in my Rust repo, find the 3 most common problems people are hitting, and write a brief status update I can add to the README."*
- Type at natural speed — don't rush

### Step 4 — Show It Working *(0:25 – 0:40)*
- Hit enter and **let the activity stream play out on screen**
- Ideal: show animated steps like "Reading issues… Grouping by theme… Writing summary…"
- **Go silent here for 3–4 seconds** — the demo sells itself
- **Say:** *"Watch this... [pause]... That's it."*

### Step 5 — Show the Finished Output *(0:40 – 0:52)*
- Show the generated markdown doc / status update on screen
- Scroll through it briefly — let the quality speak
- **Say:** *"No prompt engineering. No switching between five tabs..."*

### Step 6 — Pricing Callout + CTA *(0:52 – 1:05)*
- You can stay on the result screen or cut back to your face
- **Say:** *"It's free to start... Link is in the description. Okay — back to Rust."*

## 🦀 Demo Task Ideas

Pick the one that fits your tutorial topic best:

| Tutorial Topic | Suggested Demo Task |
|---|---|
| General Rust / systems | Summarize open GitHub issues + write status update |
| Async Rust | *"Read these async runtime error logs and document the top 3 footguns"* |
| CLI tools | *"Look at my Cargo.toml and write a summary of what each dependency does"* |
| WebAssembly | *"Read the open issues in a WASM repo and categorize them by browser vs toolchain"* |
| Error handling | *"Take these compiler errors and write a beginner-friendly explanation of each one"* |
| Memory / ownership | *"Summarize the borrow checker questions from this GitHub discussion thread"* |

## ✅ Production Checklist

### Before Recording
- [ ] MindsHub desktop app installed and open
- [ ] GitHub connected in MindsHub settings
- [ ] A Rust repo with real open issues is linked (or use a demo repo — see `/demo-data/`)
- [ ] OBS / screen recorder ready, mic tested
- [ ] Script printed or on second monitor

### During Recording
- [ ] Show the connected apps panel clearly
- [ ] Let the activity stream run — **don't cut it short**
- [ ] Hold the finished output on screen for at least 4 seconds
- [ ] Say the pricing clearly: *"five million tokens free... $9.95 Pro"*

### After Recording
- [ ] Trim to 60–70 seconds max
- [ ] Add `mindshub.ai` to the video description
- [ ] Pin the link as a top comment
